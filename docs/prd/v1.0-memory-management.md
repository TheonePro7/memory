## 1. 背景与目标

### 1.1 背景

全球 12+ 家公司在做 Agent 记忆存储（Mem0 48K⭐、Zep 24K⭐、Letta 21K⭐ 等），但没有一家在做记忆管理。Agent 记错了没人知道，记重复了没人清理，换了记忆系统数据带不走。

在人机协同时代（而非纯 AI 自治时代），人类需要：
- **看得见** Agent 在记什么
- **改得了** Agent 记错的内容
- **管得住** Agent 记忆的演化

### 1.2 产品定位

**Agent 记忆的管理层** — 一个独立于任何记忆存储产品的管理平面，让人能够看见、修正、组织和迁移 AI Agent 的记忆，无论底层用的是哪个记忆系统。

### 1.3 目标

V1.0 目标：用户能安装后在自己的 Dashboard 里**看到** Agent 记住了什么，并且能**修改或删除**错误的记忆。

## 4. 技术方案

### 4.1 架构

```
Agent → remember() / recall()  MCP
                ↓
          ┌─────────────┐
          │   同步层     │  ← core.py 新增
          │  (core.py)  │
          └──────┬──────┘
                 │
          ┌──────┴──────┐
          ↓              ↓
    ┌──────────┐  ┌──────────┐
    │ 自有存储   │  │ 第三方    │
    │ ChromaDB  │  │ Mem0 /   │
    │ + SQLite  │  │ Markdown │
    └──────────┘  └──────────┘
          ↑
    ┌──────────┐
    │ Dashboard │
    │ 浏览+编辑  │
    └──────────┘
```

### 4.2 编辑限制与裂变方案

编辑限制使用 SQLite 存储，路径 `~/.agent-memory/quota.db`：

```sql
CREATE TABLE IF NOT EXISTS usage (
    month TEXT NOT NULL,        -- "2026-05"
    operation_count INTEGER DEFAULT 0,
    PRIMARY KEY (month)
);

CREATE TABLE IF NOT EXISTS referrals (
    install_id TEXT PRIMARY KEY,   -- 推荐人 ID
    bonus INTEGER DEFAULT 0,       -- 累计加成（每推荐一人 +50）
);
```

- 基础配额：**100 次/月**
- 裂变加成：每成功推荐一人，双方永久 +50 次/月
- 推荐流程：`agent-memory refer` 输出邀请码 → 新用户 `npx @ivanston/init --refer <code>` → 双方 quota.db 各 +50
- 跨月不重置加成（只重置 usage 计数）
- 纯本地 SQLite，不需要用户认证

### 4.3 同步层设计

```python
# core.py 改动最小
def remember(content, ..., sync_third_party=True):
    # 1. 总是写自有存储
    result = mem0_backend.add(content, ...)
    # 2. 可选写第三方
    if sync_third_party and has_adapter:
        adapter.add(content, ...)
    return result
```

### 4.4 适配器接口

```python
class MemoryAdapter(ABC):
    @abstractmethod
    def list_all(self) -> list[dict]: ...
    @abstractmethod
    def get(self, id: str) -> dict | None: ...
```

适配器只实现读取接口（管理层的核心是读数据），编辑/删除走自有存储。这样适配器开发成本极低。

```

  1. npx @ivanston/init → 安装成功
  2. 打开 Dashboard → 看到"你的 Agent 记住了 N 条信息"
