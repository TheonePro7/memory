# PRD: Dashboard 总览页重设计 — 工作台入口

## 为什么做

当前 Overview 页面只有三个统计数字（记忆总数/会话总数/近7天会话），跟其他模块完全脱节。用户每次打开 Dashboard 无法快速了解项目状态，需要在各页面之间反复跳转。

目标用户：你自己（Agent Memory 的开发者/使用者），每天多次使用 Dashboard 跟踪工作进度。

## 目标

把 Overview 从"统计面板"改造成"Agent 记忆工作台"，一站式展示：
- 当前正在进行的 Agent 活动
- 最近的会话摘要（可交互）
- 待办/进行中的任务（按 Agent 分组）
- 快速导航到各功能模块

## 用户故事

1. 作为用户，我打开 Dashboard 第一眼就能看到**哪些 Agent 最近活跃过**，以及它们的状态
2. 作为用户，我能在总览页直接看到**最近的会话列表**，点击展开查看详情
3. 作为用户，我能在总览页看到**待办和进行中的任务**，按 Agent 分组展示
4. 作为用户，我能在总览页通过快捷卡片**一键跳转到**记忆/Agent/时间线/任务页面
5. 作为用户，我能在总览页看到**快速操作入口**（创建任务、搜索记忆、查看活跃任务）

## 验收标准

1. [ ] Overview 页面展示最近 7 天活跃的 Agent 列表（来源：/api/agents/scan + /api/tasks/agents）
2. [ ] Overview 页面展示最近 5 条会话摘要（来源：/api/sessions）
3. [ ] Overview 页面展示待办+进行中的任务卡片，按 Agent 分组（来源：/api/tasks?status=todo&status=in_progress）
4. [ ] 每个快捷卡片可点击跳转到对应页面
5. [ ] 页面加载时有骨架屏，错误时显示降级状态
6. [ ] TypeScript + Vite build 通过

## 设计约束

- 后端不新增 API 端点，复用现有接口
- 如果某个模块数据加载失败，不影响其他模块展示（模块级容错）
- 保持 Vercel 暗色主题风格一致
