## 定位

Dolt 本身不是 Agent 记忆项目，而是 **beads 的存储基座**。它是一个 MySQL 兼容的数据库，每行数据都是版本化的。

## 核心能力（与记忆相关）

| 能力 | 说明 | Agent 记忆价值 |
|---|---|---|
| **Git 风格版本控制** | commit/branch/merge/rebase/tag 全部支持 | Agent 的实验可分支、可回滚、可合并 |
| **Cell-level Merge** | 两方同时改不同字段不冲突 | 多 Agent 并发写入天然安全 |
| **Diff 分析** | 任意两个版本间计算差异 | 记忆变化可审计 |
| **时间旅行查询** | `SELECT ... AS OF 'commit-hash'` | 查询 Agent 任意历史状态 |
| **SQL 接口** | MySQL 兼容，标准 SQL 查询 | 不引入新查询语言 |

## 与 beads 的关系

```
beads (bd) — 版本化任务图记忆层
    ↓ 使用
Dolt — 版本化存储引擎
    ↓
beads 把每次 Agent 执行记录为一个"bead"（珠子）
珠子之间存在时序和因果关系
```
